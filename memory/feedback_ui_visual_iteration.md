name: UI visual iteration workflow
description: User loses time in visual/CSS sessions due to vague descriptions and multiple rejected attempts
type: feedback
---

Always ask for exact end state before writing any UI/CSS code. Do not try multiple techniques speculatively.

**Why:** 18 of 49 analyzed sessions were visual/UI work, many ended as "partially achieved" with multiple rejected attempts. The wave transition session is a clear example — clip-path, SVG overlays, and margin adjustments were all tried before success.

**How to apply:** When user requests a visual change, respond with a plan first and wait for confirmation. If the description is vague, ask for specifics before coding (e.g., "use a single SVG wave, white on dark background, 60px tall"). Break large visual tasks into small single-concern steps.
